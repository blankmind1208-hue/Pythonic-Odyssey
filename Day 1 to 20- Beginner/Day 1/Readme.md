Welcome to Day 1 of your Python journey! This is the most exciting step because you are officially telling a computer to do something.

Here is your mini-lesson on how to "Print into the World" and how to fix things when they go wrong.

### 1. The `print()` Function

In Python, `print()` is like a megaphone. Whatever you put inside the parentheses, the computer will shout back to you on the screen.

**The Syntax:**

Python

`print("Hello World")`

- **The Keyword:** `print` (must be lowercase).
- **The Parentheses:** `()` These tell Python that `print` is an action to perform.
- **The Quotes:** `" "` These tell Python that the stuff inside is **text** (called a "String"), not code.

### 2. String Manipulation

You can do more than just simple sentences.

- **New Lines:** Use `\n` to start a new line inside a single print statement.Python
    
    `print("Hello\nWorld") # Outputs Hello on one line and World on the next.`
    
- **Concatenation:** You can add strings together using the `+` sign.Python
    
    `print("Hello" + " " + "Angela")`
    

### 3. Debugging (The "Art of Fixing")

When you code, you will get "Bugs" (errors). Don't panic! Here are the most common ones for Day 1:

- **SyntaxError:** You forgot a closing quote `"` or a parenthesis `)`.
- **IndentationError:** You accidentally put a space at the beginning of the line. Python is very picky about spaces!
- **NameError:** You wrote `Print` instead of `print`. Python is case-sensitive.
